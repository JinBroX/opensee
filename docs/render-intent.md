# OpenSee Render Intent Layer

## 为什么 OpenSee 禁止随机渲染层

`selectVersion()` 使用 `salt32 % 100` 随机分配 V1/V2/V3。
这意味着同一个卦——同一个结构——每次可以有不同的表达方式。

这违反 OpenSee 宪法：

> 卦的结构决定一切。semantic 不能决定结构，随机也不能。

**随机 = 卦没有自己的声音。**

Render Intent 的建立，让每个卦自己决定它应该如何被表达。

## 为什么不同卦有不同的表达方式

64 卦不是 64 个"主题"。它们是 64 种**结构状态**。

不同的结构状态，需要不同的表达维度：

- **结构明确的状态**（如 Q1 乾：pure yang, clear stages）→ 需要 V1 的结构解释。"你在哪个位置"比"你感觉如何"更重要。
- **需要被容纳的状态**（如 Q29 坎：陷落）→ 需要 V2 的动作容器。在坑里的人不需要更多分析——需要知道"怎么呼吸"。
- **不适合直接解释的状态**（如 Q61 中孚：inner sincerity）→ 需要 V3 的意象共振。有些东西说透了反而假了。

## V1/V2/V3 不是"文案版本"

它们是三种**易经表达维度**：

| V | 维度 | 适用状态 | 语言特征 |
|---|------|----------|----------|
| V1 | 结构解释 | 方向明确、阶段清晰 | 判断、阶段、风险、定位 |
| V2 | 动作容器 | 渐进、调整、修复、容纳 | 轻动作、微感受、可投射 |
| V3 | 意象共振 | 感应、象征、不适合直说 | 留白、诗意、不解释 |

同一卦的 V1/V2/V3 是同一结构的不同呈现维度，不是三套不同的文案。

## Render Intent 是"卦如何开口"

每个卦的 `render-intent.json` 定义了：
- `primary`: 卦最适合用哪种维度表达
- `secondary`: 当 primary 不可用时的备选
- `reason`: 为什么选择这个维度（内部原因，不给用户看）

这不是 UI 风格选择——是**卦的结构决定了它应该如何被听见**。

## 分布统计

| V 层 | 主导卦数 | 典型卦 |
|------|---------|--------|
| V1 | 13 | Q1, Q2, Q7, Q13, Q14, Q34, Q35, Q37, Q45, Q49, Q51, Q57 |
| V2 | 43 | Q5, Q12, Q20, Q24, Q29, Q41, Q47, Q53 |
| V3 | 8 | Q30, Q31, Q44, Q50, Q58, Q59, Q61, Q64 |

### 最适合 V3 的 8 卦

Q30 离 — 火，依附，燃烧。不适合直说"你应该依附什么"。
Q31 咸 — 感应，共鸣。说出来反而浅了。
Q44 姤 — 相遇。突然出现的东西，保持它的模糊性更好。
Q50 鼎 — 容器，转化。意象本身就足够强。
Q58 兑 — 喜悦。快乐不需要解释。
Q59 涣 — 散去。让意象自己流动。
Q61 中孚 — 内在真诚。最深的东西不适合分析。
Q64 未济 — 永远在路上。未完成本身就是答案。

### 最不适合 V2 的卦

以下卦的 primary 是 V1 或 V3，且 V2 不在 secondary 中：
Q6, Q8, Q10, Q16, Q17, Q19, Q33 — 这些卦的结构需要清晰判断（V1），动作容器（V2）会淡化结构张力。
